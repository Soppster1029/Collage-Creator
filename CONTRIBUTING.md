# Contributing Guidelines

Thank you for your interest in contributing! All contributions are welcome — from bug fixes and documentation improvements to new features.

## Getting Started

1. **Fork** this repository to your own GitHub account.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/Soppster1029/Collage-Creator.git
   cd Collage-Creator
   ```
3. **Create a branch** for your changes:
   ```bash
   git checkout -b your-feature-or-fix
   ```
4. **Make your changes**, then commit them with a clear message:
   ```bash
   git commit -m "Short description of what changed"
   ```
5. **Push** your branch to your fork:
   ```bash
   git push origin your-feature-or-fix
   ```
6. **Open a Pull Request** against the `main` branch of this repository.

## Pull Request Guidelines

- Keep PRs focused — one feature or fix per PR makes review faster.
- Provide a clear description of what you changed and why.
- Link any related issues (e.g. `Closes #42`).
- Make sure your code works before submitting — include steps to test if it's not obvious.
- Be responsive to feedback; PRs with no activity may be closed after 30 days.

## Reporting Bugs

Open a [GitHub Issue](../../issues) and include:

- A clear description of the problem
- Steps to reproduce it
- What you expected to happen vs. what actually happened
- Your environment (OS, version, etc.) if relevant

## Suggesting Features

Open a [GitHub Issue](../../issues) with the `enhancement` label. Describe the use case and why it would be valuable — the more context the better.

## Reporting Security Issues

Please **do not** open a public issue for security vulnerabilities. See our [Security Policy](SECURITY.md) for how to report them privately.

## Code Style

- Follow the conventions already present in the codebase.
- Keep code readable — prefer clarity over cleverness.
- Add comments where the intent isn't immediately obvious.

## Commit Messages

Use short, descriptive commit messages in the imperative mood:

- `Fix crash when input is empty`
- `Add support for dark mode`
- `Update README with install instructions`

Avoid vague messages like `fix stuff` or `WIP`.

## Need Help?

If you're unsure about anything, open an issue and ask — we're happy to help you get your contribution across the finish line.
