# Security Policy

## Supported Versions

Only the most recent version of this project receives security updates. We strongly recommend always running the latest release.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅ Yes    |
| Older   | ❌ No     |

## Reporting a Vulnerability

We take security vulnerabilities seriously and appreciate your efforts to responsibly disclose any issues you find.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, use [GitHub's private vulnerability reporting feature](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability):

1. Go to the **Security** tab of this repository.
2. Click **"Report a vulnerability"**.
3. Fill in the details of the vulnerability and submit.

This ensures your report is visible only to the maintainers until we have addressed the issue.

### What to Include

To help us triage and resolve the issue as quickly as possible, please provide:

- A clear description of the vulnerability
- Steps to reproduce the issue
- The potential impact or attack scenario
- Any suggested mitigations or fixes (optional)

### What to Expect

- **Acknowledgement** within 48 hours of your report
- **Status update** within 7 days, including an initial assessment
- **Resolution or mitigation** as quickly as possible depending on severity
- Credit in the release notes (if desired)

We follow a **90-day disclosure timeline** — we ask that you give us up to 90 days to address the issue before any public disclosure.

## Scope

The following are considered in scope for vulnerability reports:

- Security bugs in the latest version of the codebase
- Authentication or authorization flaws
- Data exposure or injection vulnerabilities
- Dependency vulnerabilities with a direct exploitable impact

The following are **out of scope**:

- Vulnerabilities in outdated versions
- Social engineering attacks
- Denial-of-service attacks
- Issues without a clear security impact

## Thank You

We value the security community's efforts in keeping this project safe. Thank you for practicing responsible disclosure.
